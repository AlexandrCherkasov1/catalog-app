# Каталог товаров

Тестовое приложение на Next.js: каталог товаров, избранное, корзина и отдельная страница товара.

Демо: [https://catalog-app-next.netlify.app/](https://catalog-app-next.netlify.app/)

## Запуск

```bash
npm install
npm run dev
```

Локально проект открывается на `http://localhost:3000`.

## Команды

```bash
npm run dev
npm run build
npm run start
npm run lint
npm run typecheck
npm test
```

## Стек

- Next.js App Router
- TypeScript
- Redux Toolkit
- RTK Query
- SCSS Modules
- ESLint, Prettier, Husky

## Что реализовано

- каталог товаров с загрузкой из API;
- состояния загрузки, ошибки и повторного запроса;
- поиск по названию с debounce;
- фильтры по наличию, цене, категории и типу;
- сортировка по цене, названию и рейтингу;
- пагинация каталога;
- добавление товаров в корзину и избранное;
- страницы корзины и избранного;
- изменение количества товара в корзине;
- подсчет итоговой суммы;
- удаление с возможностью отмены;
- сохранение корзины и избранного после обновления страницы;
- страница товара `/product/[id]`;
- хлебные крошки и metadata для страницы товара;
- счетчики корзины и избранного в шапке;
- адаптивная верстка.

## Архитектура

Проект разделен на несколько основных слоев:

- `app` - роуты, layout, providers и store;
- `entities/product` - типы, API, Redux-логика, селекторы и UI товара;
- `widgets` - крупные блоки страниц;
- `shared/ui` - общие UI-компоненты.

Логика фильтрации, сортировки и пагинации каталога вынесена отдельно, чтобы ее можно было тестировать без React-компонентов.

## API

Товары загружаются через RTK Query из API:

```text
https://maxifoxy-testfront-96b4.twc1.net/api/products
```

В приложении запрос идет через локальный route handler `/api/products`.

Ссылки на изображения приходят из API. Домен `ohotaktiv.ru` заменяется на `img.ohotaktiv.ru`, чтобы `next/image` стабильно загружал те же файлы через доступный CDN-домен.
